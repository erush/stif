# STIF Research Methodology

## Objective

Describe team behavior through interpretable tactical features rather than predictive models.

---

## Guiding Principles

Every feature must:

- answer a soccer question
- represent a tactical behavior
- be reproducible
- be aggregated
- avoid normalization
- avoid ratios
- avoid machine learning

---

## Feature Design Workflow

Soccer Question

↓

Tactical Motivation

↓

Formal Definition

↓

SQL Implementation

↓

Validation

↓

Correlation Analysis

↓

Documentation

---

## Tactical Domains

Territory

Progression

Transition

Possession

Pressure

Chance Creation

Set Pieces

---

## Evaluation

Every feature is evaluated for:

- uniqueness

- variance

- reproducibility

- interpretability

- redundancy

---

## Design Philosophy

STIF does not engineer features for quantity.

Each feature is intended to represent a tactical concept that can be interpreted by analysts, coaches, and researchers.
